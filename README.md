# CW (Callweaver) Log Viewer

This project is a log viewer utility for Callweaver (CW) log files — typically
`cw.log` or similar. It ingests the Callweaver log file and parses it into a
readable, filterable format.

## Features

- Parse Callweaver (CW) log files
- Display logs in a user-friendly format
- Filter by date/time, message type, call ID, extensions, dialplan, and other parameters
- Visualize call progress and events

## Usage

```bash
python cw_viewer.py <logfile> [options]
```

### View All Call-IDs

```bash
python cw_viewer.py cwtrunc.txt --list-calls
```
Lists every Call-ID found in the log with start time, end time, entry count, and
the first message of the call. Useful for surveying what calls are present.

### Show Summary for a Specific Call

```bash
python cw_viewer.py cwtrunc.txt --summary --call-id C-0000004c
```
Displays a structured call-flow summary for a single Call-ID. Shows the
caller identity, destination, duration, and a chronological list of key
events (trunk setup, DTMF digits, IVR menus, dial attempts, ring/answer,
voicemail, hangup, etc.).

### Summarize All Calls

```bash
python cw_viewer.py cwtrunc.txt --summary
```
Shows call-flow summaries for every call in the log, ordered by Call-ID.

### Filter by Time Range

```bash
python cw_viewer.py cwtrunc.txt --summary --from 12:38 --to 12:40
```
Restricts output to entries (or calls) whose timestamps fall within the given
time window. Combine with `--summary`, `--list-calls`, or raw mode.

### Filter by Extension

```bash
python cw_viewer.py cwtrunc.txt --ext 102
```
Shows only log entries that mention the specified extension number. Combine
with `--summary` or a time range to narrow results further.

### Filter by Process

```bash
python cw_viewer.py cwtrunc.txt --process pbx_lua.c
```
Shows only log entries generated by the named source file. Useful for isolating
dialplan execution (`pbx_lua.c`), SIP signalling (`chan_sip.c`), queue behavior
(`app_queue.c`), and other subsystems.

### Other Options

| Flag | Description |
|------|-------------|
| `--show-noise` | Include `config.c` and `res_awstranscribe.c` lines excluded by default |
| `--year YEAR`  | Override the year used for timestamp parsing (default: 2026) |
